# 原始流水账本驱动型资产管理看板 (v5.2 - 体验与安全增强版)

本版本在 v5.1 的“清仓强断言”基础上，深度修复了前端交互性能与单文件 HTML 固有的数据生存期痛点。引入了局部 DOM 渲染引擎与硬核本地数据双向迁移（导入/导出）工具，全面提升资产记账的连续性与安全性。

---

## 🛠️ v5.2 核心问题修复与架构升级

### 1. 局部高频无痛刷新机制（修复输入市值不实时更新、卡顿问题）
* **旧版痛点**：过去为了改变市值后刷新各行收益率，系统会粗暴地执行全局重绘（Re-render DOM）。这会导致用户在输入框内打字时，每敲一个数字输入框就会失去焦点（Focus），体验极其糟糕。
* **新版优化**：引入了精准定位的 ID 局部更新机制。当用户在【当前持仓看板】中键入最新市值时，系统仅拦截并刷新顶层 KPI 看板、图表以及**当前行对应的收益率节点（`yield_node_...`）**。输入过程极度丝滑，绝不退格，且数据 100% 实时联动。

### 2. 双向零信任数据迁移机制（解决更新代码丢失数据问题）
单文件 HTML 工具在代码迭代时，直接刷新或覆盖源文件容易导致缓存冲突或默认值覆盖。为此，顶栏右侧新增了数据防丢失军械库：
* **📥 导出数据备份**：一键将当前浏览器缓存（LocalStorage）中的全部投资组合、数年流水、历史市值快照，无损打包封装成标准的 `.json` 密文文件下载至本地。
* **📤 导入数据恢复**：无论代码如何重构、重写、甚至更换电脑和浏览器，只需一键导入该 JSON 备份，全量历史数据瞬间完美复活。

### 3. 流水生命周期闭环（修复删除无效 Bug）
* 补全并修正了原始记账流水中，删除按钮（Delete Row）的动态模板字符串变量绑定错误（`${tIndex}`）。现在用户可以任意对历史错误流水进行查漏补缺和物理擦除，底层推导引擎会在万分之一秒内自动重算全局资产成本和综合胜率。

## 📂 历史追溯与调仓登记标准规范（以 A 亏损清仓换 B 为例）

进入 **【流水原始记账】** 标签页，严格按照真实发生的先后顺序，依次登记以下三笔记录：

| 步骤与动作 | 标的名称 | 大类选择 | 金额输入 (买入+,卖出-) | 全额清仓标记 |
| :--- | :--- | :--- | :--- | :--- |
| **1. 24年买入 A** | `基金A` | `基金` | `200000` | -- *(自动禁用)* |
| **2. 25年清仓 A** | `基金A` | `基金` | `-150000` *(必须负数)* | **☑️ 必须勾选** |
| **3. 26年买入 B** | `基金B` | `基金` | `150000` | -- *(自动禁用)* |

### 📈 数据推导验证结果
切换回 **【当前持仓看板】**：
1. **基金 A**：自动被打上灰色 `已结清清盘` 标签。其存续成本和市值归零。持仓回报率永远锁定并固化显示为：`结清:-25.0%`。
2. **基金 B**：显示为 `活跃持仓`，本金成本 15W。此时在其市值框中输入今天的最新净值，母组合会自动将 A 亏损的 5W 事实纳入考量，算出真正精确的综合内部回报率（XIRR 衍生模型）。

## 💾 安全迭代更新代码“两步法”

当未来 AI 为你提供更新、更强大的升级版 HTML 代码时，请务必执行以下标准化迁移流程，保障历史数据绝对安全：

1. **备份**：在旧网页上点击右上角 **【📥 导出数据备份】**，本地电脑会收到一个名为 `家庭资产数据备份_yyyy-mm-dd.json` 的安全快照文件。
2. **更替**：用最新的 HTML 代码完全覆盖并保存本地文件。
3. **还原**：双击用浏览器打开新版本的网页（此时界面为默认模板），点击右上角 **【📤 导入数据恢复】**
